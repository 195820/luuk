# 项目后续任务分析

**版本**: 1.0
**创建日期**: 2026 年 5 月 14 日
**项目**: Luuk Image Viewer

---

## Context

Luuk Image Viewer 是一个 Electron + React + TypeScript 的本地图片查看器，MVP 阶段已完成。核心功能（图片查看、缩略图网格、多库管理、收藏系统、瀑布流视图、文件夹树浏览）均已实现并经过一定测试。当前处于**后 MVP 完善阶段**。

本文档基于需求文档 (requirements.md)、功能优化路线图 (08-功能优化路线图.md) 和代码现状，梳理后续需要完成的任务。

## 一、未完成的高优先级任务 (P0)

---

### 0. 多媒体支持（视频 + 音频 + 图片统一管理）— 完整设计文档

**解决的问题**：当前的图片库中经常混有短视频和音频文件，无法查看和管理。用户希望像管理图片一样统一管理所有媒体，并且**看图片的同时能听音频**（音频播放不独占全屏，替换顶部工具栏）。

#### 0.1 需求确认汇总

| 决策点 | 用户选择 |
|--------|----------|
| ffmpeg 打包方式 | **B** — 打包到 Electron 中，用户零配置 |
| 视频格式支持范围 | **C** — 全格式支持，不支持的在查看器内提示 |
| 音频播放器位置 | 顶部工具栏区域，双击音频时替换工具栏 |
| 双击多音频行为 | **A** — 每次双击替换当前播放 |
| GIF 动图 | 最小控制：暂停/播放按钮 |
| 音频在网格中的显示 | **B+C** — 默认隐藏，顶部开关开启后底部显示独立水平滚动区域 |
| 收藏系统 | 图片+视频共用一套收藏，收藏视图可筛选"全部/图片/视频"；音频独立收藏；不做评分 |
| 视频扫描策略 | **B** — 延迟扫描，首次打开时提取完整元数据 |
| 音频缩略图 | **B** — 不生成缩略图文件，前端 CSS 渲染 |
| 不支持的视频提示 | **B** — 查看器内显示占位提示，不弹窗 |

#### 0.2 媒体类型定义

- **图片**：`.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`, `.bmp`, `.tiff`, `.tif`
- **视频**：`.mp4`, `.webm`, `.mov`, `.avi`, `.mkv`
- **音频**：`.mp3`, `.wav`, `.flac`, `.aac`, `.ogg`, `.m4a`

#### 0.3 数据库改动

```sql
ALTER TABLE images ADD COLUMN media_type TEXT DEFAULT 'image';  -- 'image' | 'video' | 'audio'
ALTER TABLE images ADD COLUMN duration REAL DEFAULT NULL;       -- 时长（秒），图片为 NULL
ALTER TABLE images ADD COLUMN codec TEXT DEFAULT NULL;          -- 编码格式（H.264, VP9, MP3 等）
```

无需新建表，字段直接加到 images 表。

#### 0.4 扫描器 (`scanner.ts`)

```
DEFAULT_EXTENSIONS 扩展为: 图片 + 视频 + 音频扩展名

扫描逻辑:
  遍历文件 → 判断扩展名
    ├── 图片 → sharp 提取 width/height/format → media_type='image'
    ├── 视频 → 记录基本元数据（文件名、大小）→ media_type='video', duration=NULL
    │   └── 延迟提取：用户首次打开时调用 ffprobe 提取 duration/codec/分辨率
    └── 音频 → ffprobe 提取 duration → media_type='audio'
```

#### 0.5 缩略图生成 (`thumbnailer.ts`)

| 媒体类型 | 缩略图策略 |
|---------|-----------|
| 图片 | 现有逻辑不变（sharp 生成 WebP） |
| 视频 | 首次打开时，调用打包的 ffmpeg 截取第 1 秒帧 → sharp 压缩为 WebP → 存入 thumbs.db |
| 音频 | **不生成缩略图文件**，前端用 CSS 渲染音频卡片（波形背景 + 文件名 + 时长） |

#### 0.6 新增组件

**AudioPlayer.tsx**（顶部音频播放器）：
- 替换 `AppHeader`（h1 标题 + 工具栏），固定在顶部
- 显示：封面（CSS 波形）+ 文件名 + 进度条 + 播放/暂停/上一首/下一首 + 音量 + 关闭按钮
- 使用 HTML5 `<audio>` 标签
- 快捷键：Ctrl+Space = 播放/暂停

**AudioCard.tsx**（底部音频区域中的单个卡片）：
- CSS 生成的波形背景 + 文件名 + 时长
- 双击 → 调用 audioStore.play()

**MediaFilter.tsx**（收藏视图筛选标签）：
- `全部 | 图片 | 视频` 三个标签
- 点击切换收藏视图显示内容

#### 0.7 新增 Store

**audioStore.ts**：
```typescript
interface AudioState {
  isPlaying: boolean
  currentAudio: { libraryId: number; imageId: number; src: string; name: string } | null
  currentTime: number
  duration: number
  volume: number
  // 方法
  play(libraryId, imageId, src): void
  pause(): void
  resume(): void
  stop(): void  // 关闭播放器，恢复顶部工具栏
  seek(time): void
  setVolume(v): void
}
```

#### 0.8 ImageViewer.tsx 修改

```
ImageViewer 内部根据 mediaType 分发渲染:
├── mediaType === 'image'  → 现有 <img> + TransformWrapper（不变）
├── mediaType === 'video'  → <video> 标签 + 播放控制栏
│   ├── 支持: 播放/暂停、进度条、音量
│   ├── 空格键 = 播放/暂停（覆盖幻灯片功能）
│   ├── CSS transform 同样作用于 <video>（旋转/翻转）
│   └── 不支持格式 → 显示占位提示 + 文件信息
└── mediaType === 'audio'  → 不进入查看器，调用 audioStore.play()
```

#### 0.9 网格视图修改

**默认行为**：
- 网格/瀑布流只显示 `media_type IN ('image', 'video')` 的文件
- 视频缩略图右上角显示时长 badge（`02:34`）
- 不支持格式的视频缩略图上叠加"不支持"图标

**"显示音频"开关**：
- 顶部加开关，开启后在底部展开独立水平滚动区域
- 显示当前文件夹的所有音频文件（`media_type='audio'`）
- 每个音频卡片双击播放

**GIF 动图**：
- ImageViewer 右上角加暂停/播放按钮

#### 0.10 收藏系统

| 收藏类型 | 收藏视图 |
|---------|---------|
| 图片 | 原有收藏视图不变 |
| 视频 | 与图片共用收藏系统，收藏视图加筛选标签（全部/图片/视频） |
| 音频 | 独立收藏视图（收藏库的独立 tab 页） |

评分只对图片有效，视频和音频不做评分。

## 七、新增特色功能建议

### 38. 智能收藏推荐

- **描述**: 根据用户浏览行为（停留时间长、放大查看、反复回看同一张图）自动标记"可能喜欢"的图片，提供一键收藏入口
- **实现思路**: 纯行为分析，不需要 AI 模型。记录查看时长、是否放大、查看次数等指标，超过阈值后标记为候选
- **预计工时**: 1-2 天

### 39. 自定义相册

- **描述**: 用户可创建多个自定义相册（类似播放列表），跨文件夹、跨库把相关图片归入一个逻辑分组，支持自定义排序、拖拽调整顺序、相册封面设置
- **与收藏的区别**: 收藏是"我喜欢这张图"（扁平聚合），相册是"这些图属于同一主题/系列"（有组织、可排序、可命名）
- **预计工时**: 2-3 天

**数据库设计**:
```sql
CREATE TABLE albums (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    cover_image_id INTEGER,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE album_items (
    album_id INTEGER NOT NULL,
    library_id INTEGER NOT NULL,
    image_id INTEGER NOT NULL,
    sort_order INTEGER NOT NULL,
    added_at TEXT DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (album_id, library_id, image_id),
    FOREIGN KEY (album_id) REFERENCES albums(id) ON DELETE CASCADE
);
```
