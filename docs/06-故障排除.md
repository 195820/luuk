## 📥 安装问题

### Electron 下载失败

**问题**: 安装依赖时 Electron 下载失败

**解决方案**:

```bash
# 项目已配置 .npmrc 使用镜像源
# 如仍有问题，手动设置镜像
npm config set electron_mirror https://npmmirror.com/mirrors/electron/

# 重新安装
rm -rf node_modules package-lock.json
npm install
```

### sharp 安装失败

**问题**: sharp 模块安装失败

**解决方案**:

```bash
# 确保使用 Node.js 22+
node -v

# 重新安装 sharp
npm rebuild sharp
```

## ▶️ 运行问题

### 应用无法启动

**问题**: 运行 `npm run dev` 后应用未启动

**排查步骤**:

1. 检查控制台错误信息
2. 检查端口是否被占用
3. 清理缓存后重试

```bash
rm -rf dist dist-electron node_modules/.vite
npm run dev
```

### 开发模式白屏

**问题**: 应用启动后显示白屏

**解决方案**:

1. 打开 DevTools 查看控制台错误
2. 检查 React 组件是否有运行时错误
3. 清理后重新启动

```bash
npm run clean
npm install
npm run dev
```

## 🚀 性能问题

### 缩略图加载慢

**问题**: 首次加载缩略图速度较慢

**说明**: 首次扫描需要生成缩略图缓存，属正常现象。后续加载会从缓存读取。

**优化建议**:

- 首次扫描后不要删除 `.ivlib` 目录
- 大库建议等待完整扫描完成

### 内存占用高

**问题**: 应用内存占用过高

**解决方案**:

```bash
# 检查缓存统计
const stats = await window.electronAPI.getCacheStats();
console.log(stats);

# 清理缓存
await window.electronAPI.clearCache();
```

**建议**: LRU 缓存上限为 200MB，如仍过高请报告 issue。

## 💾 数据库问题

### 数据库锁定

**问题**: 操作数据库时提示锁定

**解决方案**:

1. 重启应用释放锁
2. 检查是否有其他进程访问数据库

### 库数据丢失

**问题**: 重启后库数据丢失

**排查步骤**:

1. 检查数据库文件是否存在
   - 主库：`%APPDATA%\luuk\master.db`
   - 分库：`{库目录}\.ivlib\thumbs.db`

2. 检查数据库完整性
   ```bash
   # 使用 SQLite 工具检查
   ```

3. 从备份恢复（如有）

## ❓ 常见问题 FAQ

### Q: 如何添加新库？

A: 点击"管理"按钮 → 选择图片文件夹 → 自动扫描并添加

### Q: 缩略图缓存在哪？

A: 每个库的 `.ivlib/thumbs.db` 文件中

### Q: 如何清理缓存？

A: 删除对应库的 `.ivlib` 目录，重新扫描即可

### Q: 支持哪些图片格式？

A: JPG, PNG, GIF, WebP, BMP, TIFF

### Q: 如何备份数据？

A: 备份以下文件：
- `%APPDATA%\luuk\master.db`（主数据库）
- 各库的 `.ivlib\thumbs.db`（缩略图缓存）

### Q: 如何迁移数据到新电脑？

A:
1. 复制图片文件到新位置
2. 安装应用
3. 重新添加库（指向新路径）

### Q: 快捷键不生效？

A:
1. 确保焦点在应用窗口内
2. 检查是否与其他软件冲突
3. 重启应用

### Q: 图片旋转方向错误？

A: 应用会自动读取 EXIF 方向信息，如仍有问题请报告 issue。

### Q: 收藏数据如何导出？

A: 收藏数据存储在 `master.db` 中，可使用 SQLite 工具导出：

```sql
SELECT * FROM favorites;
SELECT * FROM favorite_folders;
```
