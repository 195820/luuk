## 🔧 构建准备

### 环境检查

```bash
# 检查 Node.js 版本
node -v  # 需要 22.22.0+

# 检查 npm 版本
npm -v   # 需要 10.9.4+

# 清理之前的构建
npm run clean  # 如果有的话
rm -rf dist dist-electron release
```

### 安装依赖

```bash
npm install
```

## 🏗️ 构建流程

### 开发构建

```bash
# 启动开发服务器
npm run dev
```

开发模式会自动监听文件变化并热重载。

### 生产构建

```bash
# 构建前端 + Electron 打包（生成安装包）
npm run build
```

构建完成后，安装包位于 `release/` 目录。

### 仅构建前端

```bash
# 仅构建前端（不打包 Electron）
npm run build:dir
```

适用于仅更新前端代码的场景。

### 预览构建

```bash
# 预览构建结果
npm run preview
```

## 📦 发布流程

### 版本号管理

遵循 [语义化版本](https://semver.org/) 规范：

```
主版本号.次版本号.修订号
```

- **主版本号**：不兼容的 API 变更
- **次版本号**：向后兼容的功能新增
- **修订号**：向后兼容的问题修正

### 更新版本

```bash
# 修改 package.json 中的 version 字段
```

### 更新变更日志

编辑 `CHANGELOG.md`，记录本次发布的变更。

### 创建 Release

1. 构建项目：`npm run build`
2. 在 GitHub 创建新的 Release
3. 上传 `release/` 目录中的安装包
4. 填写发布说明

## 💿 安装部署

### Windows 安装

1. 下载 `.exe` 安装程序
2. 运行安装程序
3. 按照安装向导完成安装

### 便携版

1. 下载 `.zip` 压缩包
2. 解压到任意目录
3. 直接运行 `Luuk.exe`

## 📁 构建产物说明

### release/ 目录

| 文件 | 说明 |
|------|------|
| `Luuk Setup x.x.x.exe` | Windows 安装程序 |
| `Luuk-x.x.x-win.zip` | 便携版压缩包 |
| `latest.yml` | 自动更新配置 |

### dist/ 目录

前端构建输出，包含所有静态资源。

### dist-electron/ 目录

Electron 主进程构建输出。

## 🔍 故障排除

### 构建失败

```bash
# 清理缓存
npm cache clean --force
rm -rf node_modules
npm install

# 重新构建
npm run build
```

### 依赖问题

```bash
# 使用镜像源（项目已配置 .npmrc）
# 如仍有问题，尝试手动配置
npm config set electron_mirror https://npmmirror.com/mirrors/electron/
```
