### 一、高优先级优化 (P1)

#### 1. 增量扫描优化
**现状**: 当前扫描器支持增量扫描，但仅基于文件修改时间

**优化点**:
- 使用文件修改时间 + 文件大小快速跳过未变动文件
- 添加文件夹级别的增量检测（只扫描变动的子文件夹）
- 预期提升：增量扫描速度提升 **80-90%**

**实现建议**:
```typescript
// scanner.ts 优化
async scan(): Promise<ScanResult> {
  // 增量扫描优化
  for (const filePath of imageFiles) {
    const stat = fs.statSync(filePath);
    // 快速检查：文件大小 + 修改时间
    const existingImage = this.db.getImageByPath(relativePath);
    if (existingImage && 
        existingImage.file_size === stat.size && 
        existingImage.modified_time === stat.mtime.toISOString()) {
      result.skipped++;
      continue; // 跳过未变化文件
    }
  }
}
```

#### 3. 图片预加载优化
**现状**: 有基础的缩略图预加载

**优化点**:
- 智能预加载相邻图片的原图（查看器模式下）
- 根据用户滚动速度动态调整预加载数量
- 后台预加载收藏图片

**实现建议**:
```typescript
// imageStore.ts 中添加预加载逻辑
prefetchAdjacentImages: async (currentIndex: number, direction: 'next' | 'prev') => {
  const { images, currentLibraryId } = get();
  const preloadCount = 3; // 预加载 3 张
  
  const indices = direction === 'next'
    ? range(currentIndex + 1, currentIndex + preloadCount + 1)
    : range(currentIndex - preloadCount, currentIndex);
  
  // 后台加载原图到内存缓存
  for (const index of indices) {
    if (images[index]) {
      await window.electronAPI.getImagePath(currentLibraryId, images[index].id);
    }
  }
}
```

#### 4. 离线库自动检测
**现状**: 启动时会检测库路径

**优化点**:
- 应用启动时自动检测所有库的在线状态
- 硬盘插入/移除时自动更新状态（使用 `chokidar` 监听）
- 离线库一键重新扫描

**实现建议**:
```typescript
// main.ts 中添加硬盘监听
import chokidar from 'chokidar';

const watcher = chokidar.watch('/*/', { 
  persistent: true,
  ignoreInitial: true 
});

watcher.on('add', async (path) => {
  // 检测新硬盘插入，更新库状态
  const libraries = service.getLibraries();
  for (const lib of libraries) {
    if (lib.rootPath.startsWith(path) && lib.status === 'offline') {
      service.updateLibraryStatus(lib.id, 'online');
    }
  }
});
```

   - P1: 高用户价值，近期实现
   - P2: 中用户价值，中期实现

   - 功能上线后收集用户反馈
