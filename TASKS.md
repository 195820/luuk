### 2026-02-27 - 文件夹树功能实现

**需求**: 用户需要按照子文件夹层级浏览图片，而不是所有图片混在一起。

**实现方案**:
1. **数据库层**: 在 `ThumbnailsDB` 中添加 `getFolderTree()` 方法，递归构建文件夹树
2. **服务层**: 在 `ImageService` 中添加 `getFolderTree()` 和 `getImagesByFolder()` 方法
3. **IPC 层**: 新增 `getFolderTree`、`getImagesByFolder`、`getImageCountByFolder` 处理器
4. **前端组件**: 创建 `FolderTree` 组件，支持展开/折叠和点击选择
5. **状态管理**: 在 `imageStore` 中添加 `folderTree`、`selectedFolder` 状态
6. **UI 布局**: 添加可折叠的左侧边栏，快捷键 `F6` 切换

**技术细节**:
- 路径分隔符兼容：使用正则 `/[\\/]/` 同时支持 Windows (`\`) 和 Unix (`/`) 路径
- 文件夹计数：包含子文件夹中的所有图片
- 缓存 Key：使用 `${libraryId}-${imageId}` 确保跨库隔离

**新增文件**:
- `src/components/FolderTree.tsx` - 文件夹树组件
- `src/components/FolderTree.css` - 文件夹树样式

**修改文件**:

- `src/main/ipc/library-handlers.ts` - 添加 IPC 处理器

### 2026-02-27 - Bug 修复：缩略图缓存冲突

**问题**: 切换库后缩略图显示错误（显示之前库的图片）

**根因**: 后端缓存 key 只用了 `imageId`，没有包含 `libraryId`。不同库的图片 ID 可能相同（如每个库都有图片 ID 1、2、3...），导致缓存冲突。

**修复**:
- `image-service.ts` 中 `getThumbnail` 和 `getThumbnails` 方法的缓存 key 改为 `${libraryId}-${imageId}` 格式
- 前端 `ImageGridItem` 组件添加 `libraryId` prop，确保切换库时重新加载缩略图
- 虚拟列表的 key 设置为 `${libraryId}-${rowIndex}` 和 `${libraryId}-${image.id}`

**影响**: 确保不同库的缩略图缓存独立，切换库时正确显示对应图片。
