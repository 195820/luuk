## 🎯 测试目标

### 主要目标
- 验证 **缩略图缓存系统** 功能正确性
- 验证 **多库管理模块** 功能正确性
- 验证 **文件夹树浏览功能** 功能正确性
- 验证 **库路径冲突检测** 功能正确性
- 确保核心图片查看功能稳定运行
- 发现并修复潜在 Bug，为 Phase 2 开发奠定基础

### 验收标准
- 所有 P0 级测试用例通过率 100%
- 无崩溃级缺陷
- 内存占用在合理范围内（<500MB）
- 千级图片场景下 UI 响应流畅

## 📦 测试范围

### ✅ 待测试模块

|------|--------|--------|

### 测试数据准备
```bash
# 所有测试数据均在 test-data/ 目录下

# 基础测试库
test-data/test-library/          # 100 张测试图片（基础功能测试）

# 性能测试库
test-data/large-library/         # 1000+ 张图片（性能测试）

# 多库管理测试库
test-data/multi-library-1/       # 50 张图片（多库管理测试库 1）
test-data/multi-library-2/       # 50 张图片（多库管理测试库 2）

# 文件夹树测试库
test-data/tree-library/          # 280 张图片（3 级目录结构，文件夹树测试）
```

### 生成测试数据
```bash
# 生成所有测试数据
npx tsx scripts/generate-test-images.ts all

# 或单独生成某个测试库
npx tsx scripts/generate-test-images.ts default   # 100 张基础库
npx tsx scripts/generate-test-images.ts large     # 1000 张性能测试库
npx tsx scripts/generate-test-images.ts multi     # 多库测试
npx tsx scripts/generate-test-images.ts tree      # 文件夹树测试
```

## 🧪 测试策略

### 测试类型
1. **功能测试** - 验证各功能模块正确性
2. **集成测试** - 验证模块间交互
3. **性能测试** - 验证大数据量场景
4. **回归测试** - 修复 Bug 后验证

### 测试方法
- 手动测试为主
- 关键路径编写自动化脚本（可选）
- 使用 DevTools 监控性能指标

### 测试流程
```
测试准备 → 执行测试 → 记录缺陷 → 修复验证 → 回归测试
```
